# Staff/Shift Scheduling — Run Transcript

A condensed record of the run, showing the angle gate and the synthesis path. Companion to `staff-shift-scheduling-source-notes.md` and `staff-shift-scheduling-white-space-map.md`.

## Trigger
> **User:** research scheduling tools

## Angle Gate
Clara refused to research on a vague category. "Scheduling tools" splits into categories with almost no vendor overlap, and a one-vendor read isn't intelligence. Four gate questions asked:

| Field | Answer |
|---|---|
| Category | **Staff/shift scheduling** (rostering hourly workforce — chosen over meeting-booking, appointment-booking, field-service dispatch) |
| Buyer segment | **Mid-market** (50–500+ hourly, multi-location) |
| Decision | **Product wedge** |
| Source mode | **Web-fetch public sources** |

Gate fields the user did not set — **competitor set** and **prior belief** — were defaulted: a mid-market shift-scheduling set (Deputy, When I Work, 7shifts, HotSchedules/Fourth, Homebase, Sling, Quinyx, Workforce.com), and no prior belief (clean read). Clara flagged that When I Work, Homebase, 7shifts, and Sling skew SMB, so their complaints would be weighted down unless they plausibly scale to mid-market.

## Evidence Collection
Fanned out across the vendor set, each researcher instructed to label every item by source type, recency, reviewer role/segment, and review-source label. Result:

- **Capterra** carried the run (role/size/date/tenure visible).
- **G2, TrustRadius, Gartner** all returned **403** — no incentivization labels obtainable anywhere.
- **Reddit/operator forums** were **inaccessible** for every vendor — the Tier-1 practitioner voice is missing.
- Vendor pricing pages and changelogs captured recent repricing and AI/compliance shipments.

Clara logged this as **source health: MODERATE, capped** — single-platform reliance + no labels + no forum voice + SMB-skewed reviewer base → hypothesis generation, not proof.

## Synthesis Path
1. **Clustered praise separately from complaints.** Praise collapsed into a table-stakes floor (easy scheduling, shift swaps, mobile visibility, payroll handoff, Excel replacement) — explicitly *not* where a wedge lives.
2. **Asked: what still hurts after adoption?** Complaints clustered by job-to-be-done and were tested against the evidence thresholds.
3. **Separated category gaps from vendor misses.** Per-location billing cliffs and single-location architecture recurred across 4 vendors with target-segment reviewers → segment-strong. Reporting/forecasting/compliance gating recurred across 4–6 vendors but Capterra-heavy → probable. Mobile reliability spanned 6 vendors but was judged near-table-stakes execution noise.
4. **Found the structural pattern:** the market is **barbelled** — SMB tools too thin at scale, enterprise WFM too heavy/opaque — leaving a **mid-market valley**. This was the only *Strong* finding and the hardest for incumbents to copy.

## Headline Output
**Wedge:** a multi-location-native scheduler for 50–500-employee operators combining (1) included multi-location governance, (2) blocking-level labor-law compliance, and (3) accessible dynamic forecasting — at transparent, published pricing with light implementation. Positioning: *"Enterprise-grade compliance and multi-location control, without the enterprise implementation or the SMB ceiling."*

**Caveat carried to the end:** without practitioner-forum corroboration and G2 label verification, Gaps B and C remain *probable*. The recommended next step is pasting 3–5 labeled forum threads on multi-location reporting and on compliance violations slipping through — the fastest path from probable to strong.
