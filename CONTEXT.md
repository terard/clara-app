# Clara Context

Clara turns public review material and field discussion into an ops-tooling white-space map.

## Inputs

| Source | File/Location | Section/Scope | Why |
|--------|---------------|---------------|-----|
| User | Conversation | Category, segment, wedge, competitor set, beliefs, decision | Defines the research angle |
| Identity | `identity.md` | Full file | Keeps posture and audience consistent |
| Rules | `rules.md` | Full file | Governs interrogation, source handling, and output format |
| Reference | `reference/source-weighting-framework.md` | Full file | Source credibility and evidence thresholds |
| Reference | `reference/ops-tooling-source-map.md` | Relevant category and source sections | Search starting points and source map |
| Working material | Web results or pasted reviews | Full source notes with labels | Evidence for the run |

## Process

1. Run the angle gate before researching.
2. Reject single-vendor summary requests and reframe them as category research.
3. Collect or accept source material across at least three competitors when possible.
4. Label every source by type, recency, independence, role fit, and confidence.
5. Cluster praise, complaints, and buyer workarounds by job-to-be-done.
6. Separate category-wide gaps from vendor roadmap misses.
7. Write source notes, a transcript, and a white-space map when producing a demo-quality run.

## Outputs

| Artifact | Location | Format |
|----------|----------|--------|
| Source notes | `demo/[category-slug]-source-notes.md` | Source table, evidence matrix, coverage caveats |
| Transcript | `demo/[category-slug]-transcript.md` | Conversation showing angle gate and synthesis |
| White-space map | `demo/[category-slug]-white-space-map.md` | Ranked gaps, evidence, buyer wedge, next validation |
